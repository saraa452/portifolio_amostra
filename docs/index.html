    Methodology

    How the project was built

    The workflow starts with structured sales records, applies aggregation logic in Python and delivers a static, shareable dashboard for portfolio presentation.

    1. Data preparation

    Standardized dates, validated metrics and aggregated KPIs by month, category, region, product and seller.

    2. Business analysis

    Identified revenue concentration, profitability patterns and operational opportunities from the commercial data.

    3. Storytelling layer

    Converted the analysis into an executive dashboard designed for GitHub Pages and portfolio visibility.

    Stack

    Tools used in the project

    • Python and pandas for aggregation and KPI generation
    • Chart.js for interactive data visualization
    • HTML, CSS and JavaScript for the public-facing dashboard
    • GitHub Pages as the deployment layer for the portfolio
